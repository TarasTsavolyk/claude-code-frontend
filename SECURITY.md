# Security Policy

## Supported versions

This is a configuration starter template, not a runtime dependency. Only the latest release is supported — copy the current `main` / latest tag.

## Reporting a vulnerability

Report privately via **GitHub Security Advisories** (the repo's **Security → Report a vulnerability** tab), not a public issue. Include the affected file (e.g. `.claude/settings.json`, `.github/workflows/*`) and the impact.

Because adopters copy this config into their own repos, config-level issues — over-broad permission rules, unsafe workflow patterns, leaked secrets — are in scope. We aim to acknowledge within a few days.
