# Internationalization

> Applies only when the project is internationalized (e.g. uses `vue-i18n`). Skip entirely for single-locale projects.

## Strings
- No hardcoded user-facing text in templates or scripts — every string goes through a translation key (`t('...')`). Keys are namespaced and stable; don't key by the English sentence.
- Never assemble a sentence by concatenating translated fragments — word order differs per language. Use interpolation with named placeholders instead.
- Use the library's pluralization (not manual `count === 1` branches); handle gender/context where the locale needs it.

## Formatting
- Format dates, numbers, and currency with `Intl` (or the i18n lib's wrappers) for the active locale — never hand-format.
- Keep message catalogs out of components; load them lazily per locale/route to avoid bloating the initial bundle.

## RTL & layout
- Support RTL with CSS logical properties (`margin-inline-start`/`-end`, `text-align: start`/`end`, `inset-inline-*`) instead of physical left/right — in Tailwind, the `ms-`/`me-`, `text-start`/`text-end`, `start-*`/`end-*` utilities. Drive direction from `dir` on the root.
- Don't assume text length — layouts must tolerate translations 30–40% longer than English without truncation or overflow.
